# 🪶 Contributing to the Book of Spells

*So you would add to the Book. Good. But know first how this grimoire grows — for
it is unlike other books.*

## We do not write spells. We gather them.

The Book of Spells holds no spells of our own authorship. Every spell in `skills/`
is **transcribed from another wizard's grimoire** — a public git repository kept
by some house elsewhere. We import their work; *they* keep it current. This is
deliberate: we never want to maintain a spell, only to point at one and let its
true author tend it.

So to contribute a spell, you do not write a `SKILL.md`. You **nominate a house**
— a git repo of skills — and the Rite of Transcription draws its spells into the
Book. Your contribution is a few lines in [`spellbook.toml`](./spellbook.toml).

> Want to fix the *prose grimoire* (`README.md`) or the tooling instead? That's
> welcome too — jump to [Other contributions](#other-contributions).

## What may enter the Book — the Trust Gate

A spell the Book imports is an instruction the demon will *obey*. We therefore
treat only with **accountable houses**. A nominated repo must be one of:

- **An established GitHub Organization** — either *verified*, or *old and broad*
  (≥ 5 years old and ≥ 30 public repositories). A five-minute vanity org does not
  count. Real houses: `angular`, `cypress-io`, `google`, `snyk`, `makenotion`.
- **A hand-vetted individual** on a short allowlist (currently `mattpocock`,
  `kepano`). We add to this list rarely, and only with a clear reason in the PR.

**Popularity is not a gate.** Stars measure virality, not trust — a 100k-star
repo from an unknown individual is still street magic, and stays out. The house's
*accountability* is what we trust, not its fame.

If your nomination is an individual or a young org, expect to make the case for
it. "It has many stars" is not a case.

## What the Book refuses — the Scrying Ward

We pull the latest from each house on every run, but we **scry** every incoming
spell for dark magic before it may enter (`scry()` in `transcribe-spells.py`).

- A spell carrying an **adversarial instruction** — *"ignore previous
  instructions"*, *"do not tell the user"*, secret key material being sent
  off-box — is **banished**: it never enters the Book and the run fails.
- A spell with a **suspicious-but-explicable** marker — a security skill
  discussing exfiltration, a deploy skill piping an installer to a shell — is
  **kept under watch**: admitted, but flagged for a human to glance at.

If your nominated house trips the Ward, the offending spell is named in the run
output. Usually the fix is to scope `roots`/`exclude` away from test fixtures or
examples; occasionally it means the house isn't a fit.

## How to nominate a house

1. **Find a qualifying repo** (passes the Trust Gate above) that contains
   `SKILL.md` files.

2. **Add a `[[source]]` block** to [`spellbook.toml`](./spellbook.toml):

   ```toml
   [[source]]
   name = "acme"                              # provider prefix → acme-<skill>
   url = "https://github.com/acme/skills"     # the repo to gather
   repo = "acme-skills"                       # folder under temp-repos/
   trust = "verified-org"                     # why it passes the gate (see below)
   roots = ["skills"]                         # optional: where SKILL.md live
   exclude = ["examples/*", "template-skill"] # optional: skip these
   ```

   - `name` becomes the prefix on every transcribed spell, so names never collide
     across houses (`acme-deploy`, not a bare `deploy`).
   - `trust` documents the gate verdict: `verified-org`, `established-org`,
     `org-borderline`, or `individual-allowlisted`. Be honest here — it's the
     audit trail.
   - Omit `roots` to scan the whole repo; set it to avoid pulling tests/examples.

3. **Gather and transcribe locally** (this project runs on [uv](https://docs.astral.sh/uv/)):

   ```bash
   uv run gather-reliquaries.py     # clones/refreshes temp-repos/ (gitignored)
   uv run transcribe-spells.py      # regenerates skills/ — must exit 0
   ```

   A clean run exits `0`. If it exits `2`, the Ward banished a spell or two spells
   clashed — read the output and resolve it before opening a PR.

4. **Open a pull request** with:
   - the `spellbook.toml` change, **and** the regenerated `skills/` it produced
     (commit both — `skills/` is the shared, browsable Book);
   - a sentence on the house and *why it passes the Trust Gate*.

> **Never hand-edit `skills/`.** It is wholly regenerated by the Rite of
> Transcription on every run; edits there are erased. Change `spellbook.toml` and
> re-run instead.

## Other contributions

- **The prose grimoire (`README.md`)** — adding a link or section? Read
  [`CLAUDE.md`](./CLAUDE.md) first: keep the magical register, give each entry a
  true name whose real referent is recoverable, and respect the narrative arc.
- **The tooling (`*.py`)** — improvements to gathering, transcription, the
  catalogue, or the Ward are welcome. If you touch the Ward, heed the hard-won
  rule in `CLAUDE.md`: banish on adversarial *phrasing*, never on a topic
  *mention*, or you will mutilate honest security skills.

*Sign the ledger. Mind the wax.* 🕯️
